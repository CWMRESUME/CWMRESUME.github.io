mysql基础部分

基础操作DDL

SQL语句 作用
SHOW DATABASES; 查询所有数据库
SELECT DATABASE(); 查询当前使用数据库

CREATE DATABSE [IF NOT EXISTS] 数据库名 [DEFAULT CHARSET 字符集] [COLLATE 排序规则];
DROP DATABASE [IF EXISTS] 数据库名;|删除数据库
USE 数据库名;|使用数据库

基础操作DCL

SQL语句 作用 描述
USE mysql; SELECT * FROM TABLE 查询用户 mysql中user信息保存在mysql.user表中
CREATE USER ‘用户名’@‘主机名’ IDENTIFIED BY ‘密码’; 创建用户 仅仅是创建，但没有权限，如果想让用户在任意主机访问该数据库，那么主机名写成%
ALTER USER ‘用户名’@‘主机名’ IDENTIFIED WITH mysql_native_password by ‘新密码’; 修改密码
DROP USER ‘用户名’@‘主机名’; 删除用户
SHOW GRANTS FOR ‘用户名’@‘主机名’; 查询权限
GRANT 权限列表 ON 数据库名.表名 TO ‘用户名’@‘主机名’; 授予权限 数据库名或表名使用*代表所有
REVOKE 权限列表 ON 数据库名.表名 FROM ‘用户名’@‘主机名’; 撤销权限 数据库名或表名使用*代表所有

删除外键

外键约束删除/更新行为

行为 说明
NO ACTION/RESTRICT 当在父表中删除/更新对应记录时，首先检查是狗有对应的外键，如果有则不允许删除/更新
CASCADE 当在父表中删除/更新对应记录时，如果有，则在子表中删除/更新外键在子表中对应的记录
SET NULL 当在父表中删除对应记录时，首先检查是否有对应外键，如果有则设置子表中该外键的值为null

mysql进阶部分

mysql体系结构

  • 连接层
    最上层时一些客户端和链接服务，主要完成一些类似于连接处理，授权认证的安全方案。服务器也会为安全接入的客户端验证它所具有的操作权限。
  • 服务层
    完成了大部分的核心功能服务功能，如sql接口，并完成了缓存的查询，sql的分析和优化，部分内置函数的执行。所有跨存储引擎的功能也在这一层实现，如过程，函数等。
  • 引擎层
    真正的负责了mysql中数据的存储和提取，服务器通过api和存储引擎进行通信。不同存储引擎具有不同的功能，这样我们可以根据自己的需要，来选择合适的存储引擎。
  • 存储层
    主要将系统文件存储在文件系统之上，并完成和存储引擎的交互。
